Пользовательский сценарий 1. Добавление контрагента.
Действующее лицо: Пользователь

Предусловие: Пользователь находится на странице "Список контрагентов"
Основной сценарий:
1) Пользователь нажал на кнопку "Добавить пользователя". Откроется страница с формой добавления контрагента.
2) Пользователь правильно заполняет все поля. Для добавления контрагента нажмите кнопку "Добавить", для отмены - "Назад".
	Альтернативный сценарий:
	2) Пользователь заполняет поля неверно. При нажатии на кнопку "Добавить" выводятся сообщения-ошибки заполнения данных.
	   Окно не закрывается, пользователь может исправить ошибки.
Постусловие: контрагент добавлен в справочник.

Пользовательский сценарий 2. Изменение данных контрагента.
Действующее лицо: Пользователь

Предусловие: Пользователь находится на странице "Список контрагентов"
Основной сценарий:
1) Напротив выбранного контрагента пользователь нажал на кнопку "Изменить". Откроется страница с формой изменения данных контрагента.
2) Пользователь правильно заполняет все поля. Для изменения данных контрагента нажмите кнопку "Изменить", для отмены - "Назад".
	Альтернативный сценарий:
	2) Пользователь заполняет поля неверно. При нажатии на кнопку "Изменить" выводятся сообщения-ошибки заполнения данных.
	   Окно не закрывается, пользователь может исправить ошибки.
Постусловие: данные контрагента изменены.

Пользовательский сценарий 3. Удаление контрагента в списке.
Действующее лицо: Пользователь

Предусловие: Пользователь находится на странице "Список контрагентов"
Основной сценарий:
1) Напротив выбранного контрагента пользователь нажал на кнопку "Удалить". Появляется всплывающее окно с подтверждением.
2) Для подтверждения нажмите кнопку "ОК", для отмены - "Отмена".
Постусловие: контрагент удалён, если в окне подтверждения была нажата клавиша "ОК".

Пользовательский сценарий 4. Удаление контрагента по наименованию.
Действующее лицо: Пользователь

Предусловие: Пользователь находится на странице "Список контрагентов"
Основной сценарий:
1) Пользователь нажал на кнопку "Удалить по наименованию". Появится всплывающее окно "Удаление контрагента по наименованию".
2) Пользователь вводит наименование существующего контрагента. Для удаления контрагента нажать кнопку "Удалить", для отмены - "Назад".
	Альтернативный сценарий:
	2) Пользователь вводит наименование несуществующего контрагента. 
	   Выводится сообщение-ошибка о том, что такого контрагента не существует. Окно не закрывается, пользователь может исправить наименование.
Постусловие: контрагент удалён, если нажата кнопка "Удалить".

Пользовательский сценарий 5. Возврат к странице "Справочник контрагентов".
Действующее лицо: Пользователь

Предусловие: Пользователь находится на странице "Список контрагентов"
Основной сценарий: пользователь нажал на кнопку "Вернуться". Откроется страница "Справочник контрагентов".
Постусловие: Пользователь находится на странице "Справочник контрагентов".
