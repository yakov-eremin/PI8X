1.	Вход в систему. 
Действующее лицо: пользователя приложения. 
Предусловие: пользователь находится на странице аутентификации. 
Основной сценарий: 
•	Ввести свой логин.
•	Ввести свой пароль.
Постусловие: откроется "Домашняя страница" - "Мои дисциплины".

2.	Просмотреть список групп, у который преподаватель сейчас (в текущем семестре) ведет дисциплины.
Действующее лицо: пользователь приложения.
Предусловие: пользователь успешно вошел в журнал (1).
Основной сценарий:
•	Сделать нажатие ЛКМ по названию группы.
Постусловие: Переход на страницу "Мои дисциплины". Отображение списка дисциплин на этой же странице.


3.	Просмотреть дисциплины, у который преподаватель сейчас (в текущем семестре) ведет дисциплины.

Предусловие: пользователь успешно вошел в журнал (1), пользователь находится на странице "Мои дисциплины" (2).
Основной сценарий:
•	Сделать нажатие ЛКМ по названию группы.
Постусловие: Отображение списка дисциплин на этой же странице.

4.	Просмотреть все дисциплины, которые преподаватель ведет, и на каких направлениях.
Действующее лицо: пользователь приложения.
Предусловие: пользователь успешно вошел в журнал (1).
Основной сценарий:

Постусловие: Переход на страницу "Мои дисциплины". Отображение списка дисциплин и направлений.

6.	Просмотреть программу обучения направления по дисциплине.
Действующее лицо: пользователь приложения.
Предусловие: выполнен сценарий (3) ИЛИ (4) ИЛИ (5)            (следовательно и (1) тоже)
Основной сценарий:
•	Сделать нажатие ЛКМ по названию дисцплины.

7.	Создание нового задания для студентов по дисциплине.
Действующее лицо: пользователь приложения.
Предусловие: пользователь находится на странице программы обучения по дисциплине (6).
Основной сценарий:
•	Нажать на кнопку "Новое задание", выполнится переход на новую страницу.
•	Ввести описание работы
•	Ввести задание к работе (необязательно)
•	Выбрать сроки выполнения работы (с помощью календаря) (необязательно)
•	Выбрать тип работы в выпадающем списке ("Лабораторная работа", "Самостоятельная работа", "Курсовая работа", "Коллоквиум", "Контрольная работа", "Экзамен"). Нажать на него ЛКМ. Вариант по умолчанию "Лабораторная работа"
•	Выбрать обязательность работы с помощью выпадающего меню ("Да", "Нет"). Нажать на него ЛКМ. Вариант по умолчанию "Да".
•	Нажать на кнопку «Create» чтобы применить изменения, либо на кнопку «Cancel», чтобы отменить изменения.
Постусловие: переход на предыдщую страницу и внесение изменений в программе обучения (если была нажата кнопка «Create»).

8.	Редактирование задания для студентов по дисциплине.
Действующее лицо: пользователь приложения.
Предусловие: пользователь находится на странице программы обучения по дисциплине (6). 
Основной сценарий:
•	Нажать на кнопку «Изменить» слева от названия работы в таблице, выполнится переход на новую страницу.
•	Ввести описание работы 
•	Ввести задание к работе 
•	Выбрать сроки выполнения работы (с помощью календаря) 
•	Выбрать тип работы в выпадающем списке ("Лабораторная работа", "Самостоятельная работа", "Курсовая работа", "Коллоквиум", "Контрольная работа", "Экзамен"). Нажать на него ЛКМ.
•	Выбрать обязательность работы с помощью выпадающего меню ("Да", "Нет"). Нажать на него ЛКМ. 
•	Нажать на кнопку «Apply Changes» чтобы применить изменения, либо на кнопку «Cancel», чтобы отменить изменения.
Постусловие: переход на предыдщую страницу и внесение изменений в программе обучения (если была нажата кнопка «Apply Changes»).

Предусловие: пользователь находится на странице программы обучения по дисциплине (6). 
Основной сценарий:

10.	Просмотр оценок студента.
Действующее лицо: пользователь приложения.
Предусловие: пользователь находится на странице программы обучения по дисциплине (6). 
Основной сценарий:
•	Нажать на кнопку «Перейти к оценкам»
•	Выбрать группу в выпадающем списке
•	Выбрать студента в выпадающем списке
Постусловие: отобразится таблица успеваемости студента

11. 	Внесение оценки студента
Действующее лицо: пользователь приложения.
Предусловие: пользователь просматривает успеваемость студента (10). 
Основной сценарий: 
•	Нажать ЛКМ на кнопку «Добавить строку»
•	В появившейся строке выбрать в выпадающем списке оцениваемую работу
•	Указать дату сдачи работы с помощью календаря (необязательно)
•	Ввести оценку
•	Нажать на кнопку «Сохранить» чтобы применить изменения
Постусловие: уведомление "Изменения сохранены"

14. 	Редактирование итоговых оценок 
Действующее лицо: пользователь приложения.
Предусловие: пользователь просматривает итоговые оценки группы (13). 
Основной сценарий: 
•	Если список студентов не отображается, нажать на кнопку "Автоматически внести данные о студентах". Страница обновится.
•	Нажать ЛКМ на кнопку «Редактировать»
•	Выставить оценку за аттестацию (первую и/или вторую)
•	Выставить оценку за семестр
•	Выставить оценку за экзамен
•	Выставить итоговую оценку
•	Нажать на кнопку «Сохранить», чтобы применить изменения
Все изменения опциональны.
Постусловие: уведомление "Изменения сохранены"
